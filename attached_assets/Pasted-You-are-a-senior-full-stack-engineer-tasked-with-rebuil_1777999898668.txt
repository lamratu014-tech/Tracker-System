AUTH SYSTEM
-----------------------------
Login must use:
- postcode
- password

Rules:
- Validate against local user dataset
- Store session in Zustand + AsyncStorage
- Persist login across app restarts
- Block access if not authenticated

-----------------------------
ROLE SYSTEM (STRICT ENFORCEMENT)
-----------------------------
ADMIN:
- Full access to all streams, teams, projects, milestones
- Can edit all data
- Can reassign leaders

LEADER:
- Can view all streams
- Can ONLY edit their own team
- Can create/edit projects and milestones within their team only

MEMBER:
- Can only view assigned team data
- No editing permissions
- No access to system controls

IMPORTANT:
Enforce permissions in BOTH:
- UI (hide actions)
- Logic (block unauthorized actions even if UI is bypassed)

-----------------------------
STATE MANAGEMENT (ZUSTAND ONLY)
-----------------------------
Create a central store containing:
- users
- currentUser
- streams
- selectedStream
- selectedTeam
- selectedProject

All updates must go through store actions:
- addStream
- addTeam
- addProject
- addMilestone
- updateItem
- deleteItem
- login/logout

Do NOT use local component state for core data.
